7. Клонировать свой личный удалённый репозиторий на компьютер.
8. Добавить файл через интерфейс GitHub. Подтянуть изменения в
локальный репозиторий.
Работу продолжать локально.
9. Получить историю операций для каждой из веток.
10. Просмотреть последние изменения.
11. Выполнить слияние в ветку master, разрешив конфликт (можно
использовать специальные редакторы или графический интерфейс git).
12. Удалить побочную ветку после успешного слияния.
13. Сделать изменения и зафиксировать их, оставляя комментарии,
несколько раз.
14. Сделать откат коммита.
15. Создать ветку для отчёта.
16. Начать оформлять отчёт в файле README.md (разрешены сторонние
редакторы с подсветкой синтаксиса), используя markdown синтаксис
(https://guides.github.com/features/mastering-markdown/):
 В отчёте должен быть снимки экрана консоли и сторонних программ.
Файлы снимков экрана разместить в отдельной папке.
 Лог команд (без результатов их выполнения).
При написании отчёта периодически делать коммиты, не забывать
комментировать.
17. Получить историю операций в форматированном виде (сокращённый
хэш + дата + имя автора + комментарий). Добавить её в отчёт и сделать
финальную фиксацию изменений.
18. Отправить локальные изменения в сетевое хранилище GitHub (если
делаете работу постепенно, то синхронизацию проводить в конце рабочего
сеанса)
В ЛК загрузить титульный лист и ссылку на свой репозиторий в формате PDF.
